# llm-daily

日々のLLM/AI運用ノート、プロンプト、タスクログを管理するためのリポジトリです。

## SlackからCodexに依頼するときの基本形

```text
対象リポジトリ: happinessisinthemind-hub/llm-daily
目的: 何を達成したいかを簡潔に書く
やってほしいこと: Codexに実施してほしい作業を箇条書きで書く
触ってよいファイル: 変更してよいファイルやディレクトリを書く
触らないでほしいファイル: 変更してほしくないファイルやディレクトリを書く
完了条件: どうなれば完了か、確認方法やテスト内容を書く
変更後は差分を見せてください
```

## Codex運用時の安全ルール

- 変更前に目的を明確にする
- 触ってよいファイルを指定する
- 秘密情報、APIキー、個人情報、医療・介護の機微情報はSlackに書かない
- 変更後は必ずdiffを確認する
- PRを確認してからマージする
- 大きな変更は小さなタスクに分ける

## Slack運用の役割分担

- Slack: 相談、指示、作業ログを残す場所
- Codex: GitHub上のファイル確認、修正、差分作成、PR作成を担当
- User: 指示内容の決定、diff確認、PR確認、マージ判断を担当
- GitHub: 変更履歴、PR、最終的なファイル管理を行う場所
